Introduction

The COMP 1950 website serves as a resource aid for students enrolled in the class. Currently, the website provides valuable course notes, links, and examples for students. This information is updated every week. The notes are not only useful for students reference when completing assignments, they also serve as the notes displayed during lecture in overhead view. Additionally, the website provides homework and project information, a link to contact the instructor, and additional associated links useful for students.

As students of COMP1950, the Samurai Falcons have been tasked to redesign the course website in order to better meet students needs. Our goal is to design a site that is visually appealing and has greater functionality and usability. Information will be easily accessible and searchable. The site will be a simple and easy to use reference source for students looking for course information, new updates, homework, and lecture notes.

Rationale

Layout

  • keep layout clean for easy readability and functionality
  • large headings so information and subject is clearly identified
  • Issues with resizing
  • all pages (client, main page, and lecture) follow same structure and layout to provide consistency in viewing site

HTML and CSS Coding:

  • Used bootstrap framework for rapid CSS development
  • bootstrap allow layout to be implemented via grid system

Navigation

  • top header navigation contains links to most frequently used sections
  • sticky header and navigation to allow for quick navigation between different areas of the site
  • eliminates need to scroll back up when wanting to look up notes from previous weeks

Custom Google Search:

  • allow ability to search the site for content
  • premade Google solution used so as to leverage the power of their search engine rather than reinventing from scratch

Homework Page

  • created a separate tumblr blog page for homework and updates so that an questions and comments discussion can be easily visible for students
  • pseparated from lecture notes for better organization and usability

Alternative Media Type

  • Smaller Style
    • for screen sizes less than 1024px wide
    • header and nav are no longer sticky and only at the top of the page so that things don't get too cluttered.
  • Overhead Style
    • to be displayed during lecture
    • similar to the smaller style with larger content text sizing to make it easier to read on overhead during lecture
  • Print Style
    • for students to print notes out for reference
    • Background colours removed and text is changed to all black for optimal black & white printing
    • navigation is completely removed, as it's not needed
    • keep the breadcrumb as it's useful in letting you know exactly where the printout came from.
  • Style Selector Button
    • added to allow for easy switching of styles, without the need for special browser tools.
